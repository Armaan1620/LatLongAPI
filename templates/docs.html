API Documentation

Everything you need to start working with latitude and longitude data in your applications.

Base URL

https://your-domain.example.com/api/v1

Endpoint: Convert coordinates

Return basic information about a coordinate pair.

Request

GET /api/v1/convert?lat=51.5&lng=-0.12

Query Parameters

  • lat – Latitude as a string (e.g. 51.5).
  • lng – Longitude as a string (e.g. -0.12).

Sample response

{
  "latitude": "51.5074",
  "longitude": "-0.1278",
  "address": "Westminster, London, Greater London, England, SW1A 1AA, United Kingdom",
  "city": "London",
  "country": "United Kingdom",
  "state": "England",
  "postcode": "SW1A 1AA",
  "road": "Parliament Square"
}

The API performs reverse geocoding using OpenStreetMap's Nominatim service, returning detailed address information including city, country, state, postcode, and road name when available.

Authentication

This demo implementation does not enforce authentication, but you can add API key checks in the Go handlers if you plan to deploy this publicly.

Rate limiting

No rate limiting is implemented out of the box in this clone. In a real deployment you can add middleware to apply per-IP or per-key quotas.
